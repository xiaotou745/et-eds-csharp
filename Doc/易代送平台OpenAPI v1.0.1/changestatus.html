获取订单类目信息

API功能描述

E代送隶属于易代送网络科技（北京）有限公司，基于共享经济模式，利用移动互联网LBS定位技术而搭建的同城即时短途配送服务平台。它将物流需求发起与骑士抢单响应的信息共享于手机终端，便捷地服务于平台商户和快送骑士。

E代送隶属于易代送网络科技（北京）有限公司，基于共享经济模式，利用移动互联网LBS定位技术而搭建的同城即时短途配送服务平台。它将物流需求发起与骑士抢单响应的信息共享于手机终端。

api地址

/order/ ChangeStatus

api功能描述址

目前只支持对接方取消当前订单

系统参数

请参照API简介

应用参数

应用参数主体如下:

参数名称 类型 是否必须 描述
order_no string 是 商家订单号（第三方的订单号)
status string 是 3：取消订单 目前只有3有效

调用json数据示例：

{
    "app_key": "wanda",
    "timestamp": "2015-03-11 20:42:33",
    "v": "1.0",
    "sign": "D58E3582AFA99040E27B92B13C8F2280",
    "fields": {
        "order_no": "12121212121"，"status": 3
    }
}

返回结果

返回结果结构

名称 类型 示例值 描述
Message string 成功 正确返回完成,错误返回错误代码
Status string 0 0为正常，其他见错误码说明

返回结果Json示例

{
    "Status": "0""Message": "成功"
}

错误码

错误码结构如下

错误代码表 描述
10000 签名错误
10001 未知系统错误
10002 参数错误
10004 订单不存在
20000 订单已经接入到E代送系统，无法取消订单
