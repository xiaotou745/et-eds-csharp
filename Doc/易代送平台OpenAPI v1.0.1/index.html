调用介绍

e代送系统API调用使用HTTP方式，即使用者可以按照一定格式拼装HTTP请求进行API调用。每个API提供正式环境和测试环境两个接口地址，接口地址如下：

正式环境：http://openapi.edaisong.com.cn

测试环境：http://edsapi.yitaoyun.net

系统级别调用参数

调用API需要传入系统参数和应用参数，其中应用参数根据具体的API具体入参，系统参数详细介绍如下：

参数名称 类型 是否必须 描述
app_key string 是 AppKey,由e代送统一分配
timestamp string 是 时间戳，格式为yyyy-mm-ddHH:mm:ss，例如：2015-03-11 13:52:03。
v string 是 api版本号，目前可选1.0
sign string 是 对API调用参数(除sign外)进行md5加密获得。详细参考如下所示

签名sign说明：
调用API 时需要对请求参数进行签名验证，e代送服务器也会对该请求参数进行是否合法验证。
验证方法如下：
根据参数名称（除sign及图片外）将涉及到验证的请求参数按照字母先后顺序排序:key + value .... key + value。
例如：将app_key=1,timestamp =2,v=3 排序为app_key=1,timestamp =2,v=3 ，参数名和参数值链接后，得到拼装字符串app_key1timestamp2v3。
将appSecret(密钥，由e代送统一分配)拼接到参数字符串头、尾进行md5加密后，再转化成大写，格式是：byte2hex(md5(appSecretapp_key1timestamp2v3appSecret))。
注：hex为自定义方法，JAVA中MD5是对字节数组加密，加密结果是16字节，我们需要的是32位的大写字符串。
